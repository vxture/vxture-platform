icon-dictionary.ts public
packages/design-system/src/icons/icon-dictionary.ts
语义化图标名称字典，定义所有可用图标的名称集合，是整个系统类型安全的唯一来源。
iconDictionary as const 数组：['home', 'arrow-left', 'chart', 'agent', ...] 共 60+ 图标名称
IconName 从 iconDictionary 推导：(typeof iconDictionary)[number]
🏷️
types.ts internal
packages/design-system/src/icons/types.ts
仅包含类型定义文件，从 icon-dictionary 导入 IconName，不包含任何运行时代码。
IconWeight 联合类型：thin | light | regular | bold | fill | duotone（Phosphor 6种粗细）
IconSize 语义尺寸枚举：xs | sm | md | lg | xl，也接受 number 直接传像素值
IconProps <Icon> 组件的 props interface，包含 name / size / weight / color / className / fallback
🗺️
icon-registry.ts internal
packages/design-system/src/icons/icon-registry.ts
图标注册中心，也是系统中唯一直接 import Phosphor 的文件。 维护语义化名称 → Phosphor 组件的映射表，新增图标需同时更新 icon-dictionary.ts。 按业务分类组织（导航 / 操作 / 数据 / 云服务 / 状态 / 用户 / 文件）。
iconRegistry const 对象：{ 'chart': ChartBar, 'agent': Robot, 'placeholder': Question, ... } 共 60+ 条目
satisfies Record<IconName, React.ComponentType<any>> 确保 registry 与 dictionary 严格一致，缺失或多余 key 都会 TS 报错
⚛️
Icon.tsx public
packages/design-system/src/icons/Icon.tsx
唯一对外的渲染组件，业务层只需引用此组件。 负责尺寸映射、表达式 fallback 降级，无运行时校验，永不抛出异常。
sizeMap 语义尺寸 → px 映射：{ xs:12, sm:16, md:20, lg:24, xl:32 }
Placeholder 最终兜底占位组件，name 和 fallback 都无效时渲染
Icon() 主组件：解析 size → 表达式查找 iconRegistry → fallback → 渲染，无 if/else 分支
📦
index.ts public
packages/design-system/src/icons/index.ts
模块统一导出入口。控制对外暴露的 API 面，业务层 import 的所有内容均来自这里。
export Icon 渲染组件，业务层主要使用的入口
export types IconName / IconProps，供业务层做类型标注
import { ChartBar, Robot, ... } from '@phosphor-icons/react'
External Library  ·  第三方图标库
@phosphor-icons/react
📚
@phosphor-icons/react
node_modules/@phosphor-icons/react
外部图标库，提供 1400+ SVG 图标 React 组件，支持 6种粗细变体。 仅被 icon-registry.ts 直接引用，业务层完全不感知此库的存在。 未来替换图标库只需修改 icon-registry.ts 一个文件。
ChartBar, Robot... 各类图标 React 组件，接受 size / weight / color / className props
