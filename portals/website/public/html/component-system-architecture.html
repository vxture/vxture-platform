Design System · Component System

Component System Architecture

@vxture/design-system 组件系统的设计架构、文件结构与调用关系全览。
所有组件底层基于 Shadcn/ui -> Radix UI Primitives 封装，统一行为与无障碍，业务层只感知 design-system 的组件 API。

以下以 Button 为例说明每个层级的职责与实现方式，其他组件（Input、Modal、Select 等）遵循相同结构。

Design System Layer · 组件系统核心
@vxture/design-system
🏷️
types.ts public types only
packages/design-system/src/button/types.ts — 每个组件均有同级 types.ts
每个组件的 TypeScript 类型定义独立存放。以 Button 为例：所有变体和尺寸以联合类型约束，避免业务层传入非法值。其他组件（Input、Modal 等）在各自目录下有相同结构的 types.ts。
ButtonVariant 'primary' | 'secondary' | 'outline' | 'ghost' | 'danger' — 按钮视觉变体
ButtonSize 'xs' | 'sm' | 'md' | 'lg' — 尺寸规格，对应具体 padding/font-size
ButtonProps 完整 props interface，继承原生 <button> 属性，扩展 variant / size / loading / leftIcon / rightIcon / fullWidth
🎨
buttonStyles.ts internal
packages/design-system/src/button/buttonStyles.ts — 对应各组件的 styles 文件
每个组件有独立的样式配置文件。使用 class-variance-authority (cva) 管理变体与尺寸的 Tailwind 类名组合，样式与渲染逻辑分离。所有组件遵循此模式：buttonStyles / inputStyles / modalStyles...
buttonVariants() cva 函数，根据 variant + size 组合返回对应 className 字符串。Button.tsx 调用此函数生成最终类名
sizeConfig 尺寸 → padding/font/icon-size 的映射对象，确保尺寸规格统一
⚛️
Button.tsx public
packages/design-system/src/[component]/[Component].tsx — 每个组件的渲染入口
对外唯一渲染组件。封装 shadcn/ui 的 Button 组件，处理 loading 状态、图标插槽、disabled 逻辑，将 shadcn 实现细节对业务层完全屏蔽。底层由 Radix Slot + Tailwind + cva 支撑。
Button() 主组件：合并 className → 处理 loading → 渲染 leftIcon/rightIcon → 通过 Radix Slot 输出 DOM
renderIcon() 内部函数，将 IconName 传给 <Icon> 组件，自动匹配当前 size 对应的图标尺寸
LoadingSpinner 内部子组件，loading=true 时替换 leftIcon 位置，使用 <Icon name="loading" /> 旋转动画
📦
index.ts public
packages/design-system/src/index.ts — 设计系统统一导出入口
设计系统统一导出入口，汇总所有组件的对外 API。内部实现（styles、primitives）不导出，业务层只通过此文件访问所有组件与类型。
export * Button / Input / Modal / Select / Icon / ... 所有组件统一从此导出
export types 所有组件的 Props 类型，供业务层做类型标注，如 ButtonProps / InputProps / ModalProps
shadcn/ui 组件源码 copy 进 design-system，底层依赖 @radix-ui + Tailwind
External Library · 第三方基础库
shadcn/ui + @radix-ui/react-slot
📚
shadcn/ui 当前选型
components/ui/ · 底层 @radix-ui/react-slot + Tailwind + cva
Radix UI 提供的 Slot primitive。核心能力是将 Button 的 props（样式、事件）合并到子组件上， 实现 asChild 模式—— 让 Button 渲染为 <a> 或其他元素，而不改变任何 API。无样式、无副作用，仅提供行为。
npx shadcn add 按需安装组件，源码落地到 design-system/src/components/ui/
Lyra 风格 方正锐利，深色模式出色，科技感强，匹配云服务/智能体平台定位
asChild / Slot 底层 Radix Slot 支持多态渲染，如 <Button asChild><a>...</a></Button>
