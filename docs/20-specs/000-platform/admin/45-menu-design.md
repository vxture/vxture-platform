# Vxture Admin 运营平台菜单设计

**Version**: 1.0.0  
**Last Updated**: 2026-04-28  
**Portal**: `portals/admin`  
**BFF**: `bff/admin-bff`

---

## 设计原则

- 按**业务域**组织一级菜单，不按职能角色或技术层次划分
- 每个业务域对应一类清晰的业务问题，内聚性优先
- 来源维度（自建 / 三方）作为产品的内部属性，不暴露为独立菜单
- 二级菜单对应独立的工作区页面，职责单一

---

## 完整菜单结构

### 一、运营总览

> **定位**：平台运营的全局视角入口。提供核心指标摘要、待处理事项和平台健康快照。
> 不承载深入分析，只做汇总展示和快速跳转。

| 二级菜单 | 路由              | 说明                                                                       |
| -------- | ----------------- | -------------------------------------------------------------------------- |
| 平台概览 | `/`               | 核心运营指标卡（租户数、活跃订阅、收入摘要、成本摘要）+ 各域关键数据趋势图 |
| 运营待办 | `/ops-todos`      | 待审核事项、异常告警、需人工介入的任务聚合列表                             |
| 服务健康 | `/service-health` | 各服务运行状态摘要，异常时高亮提示，详情跳转至「能力与服务」               |

---

### 二、租户与账号

> **定位**：平台租户体系的管理中心。覆盖租户生命周期、账号体系、
> 组织资质认证，以及运营平台自身的角色权限配置。

| 二级菜单 | 路由             | 说明                                                                             |
| -------- | ---------------- | -------------------------------------------------------------------------------- |
| 租户管理 | `/tenants`       | 租户创建、状态管理（启用 / 暂停 / 注销）、租户详情（基本信息、订阅、用量、账单） |
| 账号管理 | `/accounts`      | 平台账号列表，跨租户账号查询，账号状态管理                                       |
| 组织认证 | `/verifications` | 租户提交的企业资质认证审核，认证状态流转（待审 / 通过 / 驳回）                   |
| 运营角色 | `/admin-roles`   | 运营平台内部的角色定义与权限配置，与租户侧角色体系隔离                           |

---

### 三、产品与套餐

> **定位**：平台对外产品与服务方案的管理中心。覆盖基础产品能力、
> 行业业务产品方案、服务套餐策划，以及促销优惠配置。
>
> **产品能力说明**：
>
> - 按类型：**平台产品**（传统 SaaS 功能模块）/ **智能体产品**（AI-native 对话产品）
> - 按来源：**自建**（平台自研）/ **三方接入**（合作方提供，以平台品牌对外）
> - 产品能力是可组合、可授权、可计量的原子供给
> - 业务产品方案面向行业客户，将多个产品能力组织成一个服务场景
> - 服务套餐依附业务产品方案，定义 Free / Pro / 企业版等售卖版本、配额和价格

| 二级菜单     | 路由                 | 说明                                                                                                               |
| ------------ | -------------------- | ------------------------------------------------------------------------------------------------------------------ |
| 产品能力管理 | `/products`          | 管理可组合、可授权、可计量的基础产品能力，支持按类型（平台 / 智能体 / 大模型 / 数据能力）、来源（自建 / 三方）筛选 |
| 业务产品方案 | `/product-solutions` | 面向行业业务场景组合产品能力，如洪涝灾害监管、智慧法务；定义方案边界、包含产品和适用客户                           |
| 服务套餐管理 | `/service-plans`     | 业务产品方案下的服务套餐定义与发布（Free / Pro / 企业版等）；配置配额、价格、售卖状态和适用范围                    |
| 推广优惠     | `/promotions`        | 优惠码、折扣活动配置；适用产品 / 套餐范围设定；优惠核销数据入口（详情在「商业与财务」）                            |

---

### 四、商业与财务

> **定位**：平台商业交易链路的管理中心。覆盖从订阅建立、订单生成、
> 账单结算到用量计量的完整财务闭环。收入 / 成本摘要指标在「运营总览」展示，
> 本域承载操作管理与明细查询。

| 二级菜单 | 路由                     | 说明                                                                             |
| -------- | ------------------------ | -------------------------------------------------------------------------------- |
| 订阅管理 | `/subscriptions`         | 租户-套餐订阅关系管理；订阅状态（试用 / 生效 / 到期 / 取消）；手动变更与续期操作 |
| 订单管理 | `/orders`                | 订单列表与详情；支付状态追踪；异常订单处理                                       |
| 账单发票 | `/billing`               | 账单生成与核对；发票申请审核；对账管理                                           |
| 用量计量 | `/usage-metering`        | 各租户各产品用量明细；计量规则配置；用量异常告警                                 |
| 优惠核销 | `/promotion-redemptions` | 优惠码使用记录；折扣核销统计；与订单数据关联视图                                 |

---

### 五、能力与服务

> **定位**：平台 AI 基础能力和服务运行保障的管理中心。
> 覆盖模型 provider 接入、访问策略控制、技能注册，以及各服务的运行监控。
> 操作主体为平台运维 / 技术团队，变更频率低，影响范围大。
>
> **注**：服务健康的摘要视图在「运营总览」，本页为详情与操作入口。

| 二级菜单 | 路由               | 说明                                                                     |
| -------- | ------------------ | ------------------------------------------------------------------------ |
| 服务监控 | `/service-monitor` | 各服务运行状态详情；响应时间、错误率、可用性指标；告警规则配置           |
| 模型接入 | `/model-platform`  | LLM Provider 接入配置（Doubao、Claude 等）；API Key 管理；限流与超时配置 |
| 模型策略 | `/model-grants`    | 模型访问授权策略；按产品 / 租户 / 套餐级别的模型使用权限控制             |
| 技能接入 | `/skills`          | 智能体可调用技能的注册与管理；技能上下线；服务端点配置                   |

---

### 六、风控与合规

> **定位**：平台运营安全、客户反馈处理与合规审计的管理中心。
> 工单系统当前阶段在此域内管理，待客服智能体成熟后，
> 工单处理界面将独立为专属客服工作台（独立 portal），
> 本菜单保留工单概览与跳转入口。

| 二级菜单 | 路由             | 说明                                                                                              |
| -------- | ---------------- | ------------------------------------------------------------------------------------------------- |
| 工单反馈 | `/tickets`       | 用户工单列表；工单状态流转（待处理 / 处理中 / 待确认 / 关闭）；人工处理与分派；后续接入客服智能体 |
| 通知公告 | `/announcements` | 平台公告发布；定向通知（按租户 / 套餐 / 产品）；通知记录查询                                      |
| 审计日志 | `/audit-logs`    | 运营操作审计记录；关键操作追溯；按操作人 / 时间 / 对象筛选                                        |

## 变更说明（对比原菜单）

| 变更类型     | 原菜单              | 新菜单                             | 原因                                                     |
| ------------ | ------------------- | ---------------------------------- | -------------------------------------------------------- |
| 重命名       | 产品发布            | 产品能力管理                       | 职责不止发布，覆盖可组合、可授权、可计量的基础产品能力   |
| 新增         | —                   | 业务产品方案                       | 行业级服务平台需要在套餐之前定义业务场景和产品组合       |
| 重命名       | 订阅版本 / 订阅管理 | 服务套餐管理                       | 套餐依附业务产品方案，承载服务版本、配额、价格和适用范围 |
| 重命名       | 平台概览内嵌        | 服务健康（摘要）+ 服务监控（详情） | 总览只展示摘要，详情操作归能力与服务域                   |
| 重命名       | 平台运维            | 能力与服务                         | 与其他一级菜单命名风格对齐                               |
| 重命名       | 运营风控            | 风控与合规                         | 覆盖审计合规语义，不只是风控                             |
| 新增         | —                   | 推广优惠                           | 原菜单缺失，套餐商业化配套能力                           |
| 新增         | —                   | 优惠核销                           | 推广优惠的数据视角，挂在财务域                           |
| 新增         | —                   | 服务监控                           | 服务健康从摘要升级为可操作的监控详情页                   |
| 移除独立菜单 | 三方 Agent          | 产品能力管理内的来源属性           | 三方产品统一纳入产品能力目录，来源为内部属性             |
| 移除独立菜单 | AI 能力（一级）     | 拆入产品与套餐 + 能力与服务        | 产品运营和基础设施分属不同域                             |
| 移除独立菜单 | 计量与财务（一级）  | 并入商业与财务                     | 用量是财务前置数据，合并符合任务流                       |
| 收入/成本    | 独立菜单项          | 平台概览内的摘要指标               | 不做深入分析时无需独立菜单                               |

## 后续演进路径

**工单独立**（待客服智能体接入后）

```
当前：风控与合规 / 工单反馈
演进：独立客服工作台 portal（portals/support 或 agent-studio/cs-agent）
      风控与合规保留工单概览 + 跳转入口
```

**开发者平台**（后续规划）

```
技能接入的注册/审核环节 → 迁移至开发者平台
能力与服务保留已上线技能的运营管理视角
```

**三方接入管理**（规模化后）

```
产品能力管理内的三方来源产品数量增多后
可在产品与套餐下新增「合作伙伴」二级菜单
管理合作方资质、接入协议、结算配置
```
