来函观察"平台 `src/index.ts` 根 barrel 没有列出 `buildHealthIdentity`（只有 dist 导出）"——**观察对象是当时的 main**；该助手彼时在未合并的 PR #108（`feat/health-endpoint-contract`）上。现状：

- PR #108 已合并（main = `8f1cd9d8`），根 barrel `packages/shared/shared/src/index.ts` **显式列出** `serviceIdentity` / `buildHealthIdentity` + 两个类型导出；
- 发布产物实证（合并后本地构建验证）：`dist/index.cjs`（CJS）与 `dist/index.mjs`（ESM）运行时导出均为 `function`，`dist/index.d.ts` 含类型。

即：`import { buildHealthIdentity } from "@vxture/shared"` 在 1.5.0 上直接可用，无需任何"补 src 对齐 dist"动作。

## 3. 使用方式（引用标准 025，不在此复制）

权威：`docs/10-standards/025-service-health-endpoint-contract.md`（随 #108 已入 main）。要点：

- NestJS：`@Get("healthz")` 返回 `buildHealthIdentity({ service: "<svc>" })`；Next：`/api/health` route 同理（标准内含两段可复制样例）；
