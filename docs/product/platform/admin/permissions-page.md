# 权限管理首页设计规范

## 1. 页面定位

权限管理首页只负责展示和维护"系统权限"。

本页不处理：

- 用户绑定
- 角色绑定
- 权限分配矩阵
- 用户角色关系

本页只处理：

- 权限展示
- 权限新增
- 权限编辑
- 权限删除
- 权限启用 / 停用
- 系统预置权限保护

---

## 2. 设计目标

参考 Stripe 的清晰、专业、低干扰风格，但不采用容器嵌套。

核心要求：

- 开放式结构
- 无多层卡片嵌套
- 用缩进体现权限层级
- 支持菜单 / 按钮 / 接口权限统一展示
- 系统预置权限可识别、可保护
- 管理操作清晰克制

---

## 3. 页面整体结构

```txt
权限管理
管理平台内所有菜单、按钮和接口权限。系统预置权限用于核心功能，不建议随意修改。

[搜索权限]  [类型筛选：全部 / 菜单 / 按钮 / 接口]  [状态筛选]  [+ 新增权限]

权限结构
────────────────────────────────────────────────────────
名称                     Code                   类型     状态     来源       操作
平台管理                 platform               菜单     启用     系统预置   编辑
  用户管理               user                   菜单     启用     系统预置   编辑
    查看用户             user.view              按钮     启用     系统预置   编辑
    创建用户             user.create            按钮     启用     系统预置   编辑
    删除用户             user.delete            按钮     停用     系统预置   编辑
  角色管理               role                   菜单     启用     系统预置   编辑
    查看角色             role.view              按钮     启用     系统预置   编辑
    更新角色             role.update            按钮     启用     系统预置   编辑
系统接口
  用户登录               user.login             接口     启用     系统预置   编辑
  管理员查看             admin.view             接口     启用     系统预置   编辑
```

---

## 4. 核心展示方式

### 4.1 开放式列表

不要使用：

- 卡片套卡片
- 折叠面板套折叠面板
- 复杂树组件外观

使用：

- 单一列表
- 横向字段对齐
- 左侧缩进表示层级
- 行 hover 显示操作

---

### 4.2 层级缩进规则

```txt
一级权限：无缩进，字体略重
二级权限：左缩进 24px
三级权限：左缩进 48px
四级权限：左缩进 72px
```

示例：

```txt
平台管理
  用户管理
    查看用户
    创建用户
    删除用户
```

---

## 5. 权限类型

权限类型固定为三类：

```txt
MENU    菜单权限
BUTTON  按钮权限
API     接口权限
```

展示名称：

```txt
菜单
按钮
接口
```

颜色建议：

```txt
菜单：中性色标签
按钮：蓝色弱标签
接口：紫色弱标签
```

---

## 6. 权限字段设计

权限管理首页建议展示以下字段：

```txt
name          权限名称
code          权限编码
type          权限类型：MENU / BUTTON / API
parent        上级权限
status        启用 / 停用
source        系统预置 / 自定义
sort          排序
updated_at    更新时间，可选
```

首页默认展示：

```txt
名称 / Code / 类型 / 状态 / 来源 / 操作
```

---

## 7. 系统预置权限规则

系统预置权限是平台内置权限，默认由系统初始化生成。

### 7.1 系统预置权限允许

- 查看
- 编辑名称
- 编辑 i18n key
- 编辑描述
- 调整排序
- 启用 / 停用（需限制核心权限）

### 7.2 系统预置权限禁止

- 修改 code
- 修改 type
- 删除
- 改成自定义权限

### 7.3 核心权限保护

以下类型权限应加强保护：

```txt
登录权限
平台最高管理权限
角色管理权限
权限管理权限
租户管理核心权限
```

停用核心权限时必须二次确认。

---

## 8. 自定义权限规则

自定义权限允许：

- 新增
- 编辑
- 删除
- 启用 / 停用
- 调整父级
- 调整排序

删除前必须校验：

```txt
是否存在子权限
是否已被角色引用
是否存在接口绑定
```

如存在引用，不允许直接删除，只允许停用。

---

## 9. 新增权限交互

点击"新增权限"后，右侧抽屉或弹窗打开。

字段：

```txt
权限名称       必填
权限编码       必填，自定义权限可编辑，保存后不建议修改
权限类型       必填：菜单 / 按钮 / 接口
上级权限       可选
i18n Key       可选
描述           可选
排序           可选
状态           默认启用
```

新增规则：

```txt
菜单权限可以作为父级
按钮权限一般挂在菜单下
接口权限可以挂在菜单下，也可以独立归类在"系统接口"下
code 全局唯一
```

---

## 10. 编辑权限交互

点击某一行"编辑"。

系统预置权限：

```txt
可编辑：名称 / i18n key / 描述 / 排序 / 状态
不可编辑：code / type / source
不可删除
```

自定义权限：

```txt
可编辑：名称 / i18n key / 描述 / 排序 / 状态 / 上级权限
谨慎编辑：code / type
可删除：无引用时允许删除
```

---

## 11. 删除权限交互

删除只针对自定义权限。

删除前检查：

```txt
1. 是否有子权限
2. 是否已绑定角色
3. 是否已绑定接口路由
4. 是否存在审计记录引用
```

处理规则：

```txt
无引用：允许删除
有引用：禁止删除，建议停用
系统预置：禁止删除
```

---

## 12. 状态规则

权限状态：

```txt
enabled   启用
disabled  停用
```

权限停用后：

```txt
不再参与权限计算
保留历史绑定关系
保留审计记录
前端列表显示为灰色弱化状态
```

如果父级权限停用：

```txt
子权限不自动删除
权限计算时父级停用可导致子权限不可见或不可达
UI 上提示：上级权限已停用
```

---

## 13. 搜索与筛选

顶部提供轻量筛选：

```txt
搜索：按名称 / code 搜索
类型：全部 / 菜单 / 按钮 / 接口
状态：全部 / 启用 / 停用
来源：全部 / 系统预置 / 自定义
```

不要做复杂高级筛选。

---

## 14. 空状态

无权限时显示：

```txt
暂无权限
系统权限用于控制菜单、按钮和接口访问。

[新增权限]
```

搜索无结果时显示：

```txt
未找到匹配权限
请调整关键词或筛选条件。
```

---

## 15. 视觉规范

整体风格：

- 浅色背景
- 开放式列表
- 少边框
- 弱分割线
- 高信息密度但不拥挤

行设计：

```txt
默认：白底 / 透明底
hover：浅蓝或浅灰背景
选中：左侧细色条或浅蓝底
停用：文字灰化
系统预置：使用轻量标签
```

## 16. 推荐页面文案

页面标题：

```txt
权限管理
```

页面说明：

```txt
管理平台内所有菜单、按钮和接口权限。系统预置权限用于核心功能，不建议随意修改。
```

新增按钮：

```txt
新增权限
```

系统预置标签：

```txt
系统预置
```

自定义标签：

```txt
自定义
```

核心权限停用提示：

```txt
该权限属于系统核心权限，停用后可能影响平台正常访问。请确认是否继续。
```

删除失败提示：

```txt
该权限已被角色或接口引用，不能删除。你可以选择停用该权限。
```

---

## 17. 一句话设计原则

用开放式缩进列表展示权限结构；用系统预置和自定义区分权限来源；用停用替代危险删除；让权限管理首页只专注权限本身。
