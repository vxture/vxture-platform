## 为什么要留档（2026-07-05 教训）

这批文件 2026-07-04 曾以 untracked 状态放在 `docs/` 下，吸纳进设计文档后被直接删除
（未 commit，git 无痕）。事后稽查发现**吸纳有损**：`tenant_branding` 整表丢弃、
`tenant_contacts` 1:N 被折叠为 4 个列，且均无落盘记录——靠会话转录才恢复原稿。

**硬规**：吸纳输入稿前，先把原稿 commit 入本目录留档，并产出「吸纳对账清单」
（每个实体 → 去向 / 取舍理由）写进目标设计文档；对账完成前不得删除原稿。
